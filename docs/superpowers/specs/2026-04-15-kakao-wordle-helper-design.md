# 카카오 단어맞추기 도우미 — 설계 문서

작성일: 2026-04-15

---

## 개요

카카오 단어맞추기 게임(한국어 Wordle)을 풀기 위한 웹 도우미 앱.
사용자가 도우미의 추천 단어를 카카오 게임에 입력하고, 결과(색상 힌트)를 다시 도우미에 입력하면 도우미가 후보를 좁혀가며 최종 정답을 맞출 수 있도록 안내한다.

### 이모지 사용 금지

UI, 코드, 커밋 메시지 전체에서 이모지를 사용하지 않는다. 아이콘이 필요한 경우 lucide-react 등 아이콘 라이브러리 사용.

---

## 게임 규칙 이해

- 자음과 모음의 합이 5개인 한국어 사전 단어를 맞추는 게임
- 최대 5번의 시도 가능
- 각 시도 후 자모별 색상 힌트 제공:
  - **초록**: 해당 자모가 정답 단어의 같은 위치에 존재
  - **노란**: 해당 자모가 정답 단어에 존재하지만 위치가 다름
  - **회색**: 해당 자모가 정답 단어에 없음

## 단어 사전

### 구성 전략 (혼합 방식)

1. **기반 목록**: 국립국어원 등 공개 한국어 사전에서 자모 5개짜리 단어 추출
2. **우선 목록**: 커뮤니티(나무위키 등)에서 수집한 카카오 단어맞추기 실제 정답 단어
3. **중복 제거 후 병합**: 우선 목록 단어에 가중치 부여 (추천 우선순위)

### 자모 분해 기준

한글 유니코드 기반 분해. 음절 → 초성 + 중성 + (종성).

```
가방 → ㄱ ㅏ ㅂ ㅏ ㅇ (5 자모) ✓
나라 → ㄴ ㅏ ㄹ ㅏ (4 자모) ✗
한국 → ㅎ ㅏ ㄴ ㄱ ㅜ ㄱ (6 자모) ✗
```

파일 위치: `src/data/word-list.json`

## 핵심 알고리즘

### 힌트 필터링 (`solver.ts`)

입력된 힌트(초록/노란/회색)를 기반으로 후보 단어 목록을 필터링.

- **초록**: 해당 위치(인덱스)에 해당 자모가 있는 단어만 유지
- **노란**: 해당 자모가 단어에 포함되어 있지만, 해당 위치에는 없는 단어만 유지
- **회색**: 해당 자모가 단어에 없는 단어만 유지 (단, 같은 자모가 노란/초록으로도 표시된 경우 예외 처리)

### 엔트로피 기반 추천

남은 후보 단어 각각에 대해 기대 정보 이득(엔트로피)을 계산.

```
score(word) = -Σ p(pattern) * log2(p(pattern))
```

- 각 후보 단어를 정답이라 가정했을 때 나올 수 있는 힌트 패턴 분포 계산
- 패턴이 가장 고르게 분산되는 단어 = 정보 이득 최대 = 추천 단어

후보 수가 많을 때(>100개) 계산 비용이 있으므로 Web Worker 또는 useMemo로 처리.

## UI 구조

### 화면 레이아웃 (A 방식 — 단일 행 입력)

```
┌─────────────────────────────┐
│  카카오 단어맞추기 도우미       │  ← 헤더
├─────────────────────────────┤
│  진행 기록                   │  ← 이전 시도들 (자모 + 색상)
│  1. [ㄱ초][ㅏ노][ㅂ회][ㅏ초][ㅇ회]  가방
├─────────────────────────────┤
│  2번째 시도 추천              │  ← 현재 추천 단어 + 남은 후보 수
│  나라  (후보 84개)            │
├─────────────────────────────┤
│  결과 입력                   │  ← 추천 단어 자모 5칸
│  [ㄴ][ㅏ][ㄹ][ㅏ][_]         │    클릭 → 회색→노란→초록 순환
│  [결과 확인 → 다음 추천]       │
└─────────────────────────────┘
```

### 컴포넌트 목록

| 컴포넌트 | 역할 |
|----------|------|
| `page.tsx` | 전체 상태 관리, 게임 로직 조율 |
| `AttemptHistory` | 이전 시도 목록 렌더링 |
| `RecommendationPanel` | 현재 추천 단어 + 남은 후보 수 표시 |
| `HintInputGrid` | 5칸 자모 입력 (클릭으로 색상 순환) |
| `HintCell` | 개별 자모 칸 (회색/노란/초록 상태) |

### 색상 순환 규칙

클릭할 때마다: `회색(기본) → 노란 → 초록 → 회색` 순환.

### 이모지 없음

모든 레이블은 텍스트로. 아이콘 필요 시 lucide-react 컴포넌트 사용.

## 상태 설계

```typescript
type HintColor = 'gray' | 'yellow' | 'green'

interface Attempt {
  word: string          // 입력한 단어 (예: '가방')
  jamos: string[]       // 분해된 자모 (예: ['ㄱ','ㅏ','ㅂ','ㅏ','ㅇ'])
  hints: HintColor[]    // 각 자모의 힌트 색상
}

interface GameState {
  attempts: Attempt[]           // 지금까지의 시도 기록
  candidates: string[]          // 현재 남은 후보 단어 목록
  suggestion: string            // 현재 추천 단어
  currentHints: HintColor[]     // 입력 중인 힌트 (5칸)
  status: 'playing' | 'won' | 'lost'
}
```

## 사용자 흐름 (전체)

1. 앱 접속 → 엔트로피 계산 → 첫 추천 단어 표시
2. 사용자가 카카오 게임에서 추천 단어 입력
3. 도우미에서 결과 5칸 클릭 (색상 입력)
4. "결과 확인" 버튼 → 힌트 필터링 → 후보 재계산 → 다음 추천
5. 5번 반복 또는:
   - 모두 초록 → 정답 화면
   - 5회 소진 → 남은 후보 단어 목록 표시
